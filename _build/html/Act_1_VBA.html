Es una prueba de ra칤z unitaria que eval칰a la fuerza de la tendencia en una componente de la serie temporal

M칠todo An치litico (Prueba Dickey Fuller aumentada ADF)

Es una prueba de ra칤z 칰nica que detecta estad칤sticamente la presencia de conducta tendencial estoc치stica en las series temporales de las variables mediante un contraste de hip칩tesis.

En presencia de una autocorrelaci칩n, es necesario diferenciar la serie del precio de cierre de USDT. por tal motivo las hip칩tesis a plantear son:

H0 : 픠 =0 La serie tiene una ra칤z unitaria y no es estacionaria.

H1 : 픠 <0 La serie no tiene una ra칤z unitaria y es estacionaria.

En otras palabras, la hip칩tesis nula es la presencia de la ra칤z unitaria o la no estacionariedad mientras que la hip칩tesis alternativa sugiere la estacionariedad de los datos.

adf_result = adfuller(data["Adj Close"].tolist())

print('p-value: %f' % adf_result[1])

Valor p se define como la probabilidad de que un valor estad칤stico calculado sea posible dada una hip칩tesis nula cierta.

  • Idealmente, un valor m치s negativo del estad칤stico ADF representar치 una se침al estacionaria, en este caso de Binance (-1.131586), podria NO ser Estacionaria

  • Dado que 洧녷-value= 0.549110 > 0.05. de Adfuller es mayor que un determinado valor alfa especificado (5%, nivel de significaci칩n o intervalo de confianza del 95%), No se rechaza la hip칩tesis nula (洧냩洧녶) es decir, la serie temporal No es estacionaria.

Criterios AIC, BIC y HQIC#

Los criterios de informaci칩n de Akaike (AIC), Bayesiano (BIC) y de Hannan-Quinn (HQIC) utilizan el m칠todo de estimaci칩n de m치xima verosimilitud (log-verosimilitud) de los modelos como medida de ajuste. Estas medidas buscan valores bajos para indicar un mejor ajuste del modelo a los datos.

n_data = len(data["Adj Close"])  # Usa el nombre exacto de la columna con corchetes
n_test = 30
train_size = n_data - n_test

train = data["Adj Close"][:train_size]

def best_model(train):
    # Inicializar los mejores valores
    best_aic = np.inf  # Mejor AIC
    best_bic = np.inf  # Mejor BIC
    best_hqic = np.inf  # Mejor HQIC

    best_order_aic = None  # Mejor orden para AIC
    best_order_bic = None  # Mejor orden para BIC
    best_order_hqic = None  # Mejor orden para HQIC

    best_mdl_aic = None  # Mejor modelo basado en AIC
    best_mdl_bic = None  # Mejor modelo basado en BIC
    best_mdl_hqic = None  # Mejor modelo basado en HQIC

    # Rango de p, d y q
    pq_rng = range(5)
    d_rng = range(3)

    # Iterar sobre todos los posibles valores de p, d, q
    for p in pq_rng:
        for d in d_rng:
            for q in pq_rng:
                try:
                    # Ajustar el modelo ARIMA
                    tmp_mdl = ARIMA(train, order=(p, d, q)).fit()

                    # Obtener los valores de AIC, BIC y HQIC
                    tmp_aic = tmp_mdl.aic
                    tmp_bic = tmp_mdl.bic
                    tmp_hqic = tmp_mdl.hqic

                    # Comparar el AIC
                    if tmp_aic < best_aic:
                        best_aic = tmp_aic
                        best_order_aic = (p, d, q)
                        best_mdl_aic = tmp_mdl

                    # Comparar el BIC
                    if tmp_bic < best_bic:
                        best_bic = tmp_bic
                        best_order_bic = (p, d, q)
                        best_mdl_bic = tmp_mdl

                    # Comparar el HQIC
                    if tmp_hqic < best_hqic:
                        best_hqic = tmp_hqic
                        best_order_hqic = (p, d, q)
                        best_mdl_hqic = tmp_mdl

                except:
                    continue

    return {
        'AIC': {'best_aic': best_aic, 'best_order': best_order_aic, 'best_model': best_mdl_aic},
        'BIC': {'best_bic': best_bic, 'best_order': best_order_bic, 'best_model': best_mdl_bic},
        'HQIC': {'best_hqic': best_hqic, 'best_order': best_order_hqic, 'best_model': best_mdl_hqic}
    }

# Usar la funci칩n con tus datos de entrenamiento

Modelo ARIMA con rolling forecast#

Ahora realizaremos pron칩sticos utilizando el m칠todo de pron칩stico continuo o rolling forecasting, el cual aprovecha los datos hist칩ricos para generar predicciones continuas sobre un per칤odo. Este enfoque permite hacer ajustes en tiempo real, optimizar la planificaci칩n y mejorar la toma de decisiones. En nuestro caso, aplicaremos este m칠todo para prever horizontes de 7, 14, 21 y 28 d칤as, para cada uno de los criterios.

def arima_rolling(history, test, best_order):

    predictions = []  # Lista donde se almacenar치n las predicciones

    # Recorrer el conjunto de prueba
    for t in range(len(test)):
        # Crear y ajustar un modelo ARIMA con los datos hist칩ricos
        model = ARIMA(history, order = best_order)
        model_fit = model.fit()

        # Generar pron칩stico para el siguiente valor (1 paso adelante)
        output = model_fit.forecast()
        yhat = output[0]  # El pron칩stico es el primer valor del forecast

        predictions.append(yhat)  # Almacenar la predicci칩n en la lista

        # Obtener el valor real observado en el conjunto de prueba
        obs = test[t]

        # A침adir la observaci칩n actual al conjunto de datos hist칩ricos
        history.append(obs)

        # Imprimir el valor pronosticado y el valor real
        print('predicho = %f, esperado = %f' % (yhat, obs))

    return predictions

Modelo ARIMA sin rolling forecast#

Para realizar el pron칩stico sin el enfoque rolling, simplemente debemos eliminar la l칤nea de c칩digo que actualiza el conjunto history con los valores observados del test. Esto implica que, en la nueva funci칩n, el conjunto de entrenamiento permanecer치 est치tico y no se ajustar치 con cada nueva predicci칩n.

def arima_sin_rolling(test, modelo):
    # Calcular el n칰mero de pasos de predicci칩n basados en el tama침o del conjunto de prueba
    forecast_steps = len(test)
    
    # Realizar las predicciones para el n칰mero de pasos determinado
    forecast_tipo = modelo.forecast(steps=forecast_steps)
    
    # Convertir a un array para acceder por 칤ndice num칠rico
    forecast_values = forecast_tipo.values
    
    # Imprimir los valores observados y predichos
    for i in range(len(test)):
        observado = test[i]  # Valores observados
        predicho = forecast_values[i]  # Acceder a los valores predichos por 칤ndice
        print(f'Predicho: {predicho}, Observado: {observado}')

    # Devolver las predicciones
    return forecast_values

def forecast_accuracy(forecast, actual, str_name):

    mape = np.mean(np.abs(forecast - actual)/np.abs(actual)) # MAPE
    mae = np.mean(np.abs(forecast - actual))                 # MAE
    rmse = np.mean((forecast - actual)**2)**.5               # RMSE
    mse = np.mean((forecast - actual)**2)                    # MSE

    df_acc = pd.DataFrame({'MAE': [mae],
                           'MSE': [mse],
                           'MAPE': [mape],
                           'RMSE': [rmse],
                           'R2': [r2]},
                          index=[str_name])

    return df_acc
